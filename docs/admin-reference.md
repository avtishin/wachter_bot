# Admin Reference

## Начало работы

1. Добавить бота в чат
2. Написать в чате сообщение с `#whois` длиной ≥ 20 символов (бот запомнит вас как участника этого чата)
3. Написать боту в личку `/start`
4. Выбрать чат → настроить через меню

## Команды

| Команда | Где | Доступ | Описание |
|---|---|---|---|
| `/start` | личка | все | Открыть панель настройки чата |
| `/skip` | чат (ответ на сообщение) | только admin | Отменить таймер кика. Пользователь **не** записывается в БД — при следующем входе снова попросят представиться |
| `/approve` | чат (ответ на сообщение или на приветствие бота) | только admin | Одобрить пользователя навсегда. Записывается в БД — при следующем входе будет приветствоваться как знакомый |
| `/whois @username` | чат | все участники | Показать сохранённое представление пользователя. Также: `/whois <user_id>` или ответ на сообщение |
| `/help` | любой | все | Справка по боту |

## Настройки чата

Все настройки меняются через `/start` → выбрать чат → нажать кнопку.

| Настройка | Описание | Значение по умолчанию |
|---|---|---|
| Таймаут кика | Минут до кика (0 = не кикать, только записывать #whois) | `0` |
| Напоминание за | Минут до кика — когда отправить предупреждение (`notify_delta`) | `10` |
| Мин. длина #whois | Минимальная длина сообщения с тегом (включая тег) | `20` |
| Длительность бана | Минут (0 = навсегда) | `1` |
| Сообщение при входе | Приветствие новому участнику | `Пожалуйста, представьтесь... У вас есть %TIMEOUT%.` |
| Сообщение при перезаходе | Для уже представлявшихся | `Добро пожаловать. Снова` |
| Сообщение после представления | Ответ на успешный #whois | `Добро пожаловать.` |
| Сообщение напоминания | За `notify_delta` минут до кика | `%USER_MENTION%, пожалуйста, представьтесь...` |
| Напоминание написать #whois | Когда пользователь пишет без тега при активном таймере | `%USER_MENTION%, напишите сообщение с тегом #whois (минимум %MIN_LENGTH% символов)...` |
| Сообщение при выходе | При уходе участника из чата (`false` или `0` — не отправлять) | `%USER_MENTION% покинул чат` |
| Сообщение после кика | Публичное сообщение о кике (`false` или `0` — не отправлять) | `%USER_MENTION% молчит и покидает чат` |
| Сообщение при бане (regex) | После бана за срабатывание regex-фильтра | `%USER_MENTION%, вы были забанены т.к ваше сообщение содержит репост или слово из спам листа` |
| Regex фильтр | Регулярное выражение для фильтрации сообщений (`%TURN_OFF%` — отключить) | выключен |
| Только для новых | Применять regex-фильтр только к непредставившимся (`true`/`1` — включить) | `false` |

## Плейсхолдеры в шаблонах сообщений

| Плейсхолдер | Что подставляется | Где доступен |
|---|---|---|
| `%USER_MENTION%` | Упоминание пользователя (`@username` или ссылка на профиль) | все сообщения |
| `%TIMEOUT%` | Таймаут кика (`"15 мин."`) или `"не установлен"` если 0 | приветствие, напоминание |
| `%MIN_LENGTH%` | Минимальная длина #whois в символах | напоминание написать #whois |

Пример:
```
Привет, %USER_MENTION%! У тебя есть %TIMEOUT%, чтобы написать #whois (минимум %MIN_LENGTH% символов).
```

## Требования к #whois

- Минимальная длина сообщения: настраивается (по умолчанию **20 символов**, включая сам тег)
- Тег `#whois` должен быть распознан как hashtag entity Telegram
- Работает только в групповых чатах (не в личке)
- Принимается также **отредактированное** сообщение с тегом
- Если пользователь пишет повторно — whois обновляется в БД
