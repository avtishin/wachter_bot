# Flows

## Новый участник

```
Пользователь вступает в чат
    │
    ▼
on_new_chat_member()
    ├─ Пользователь уже в БД (ранее представлялся)?
    │       └─ да → отправить on_known_new_chat_member_message, завершить
    │
    ├─ on_new_chat_member_message == "%SKIP%"?
    │       └─ да → ничего не отправлять, завершить
    │
    ├─ Отправить приветствие (с заменой %USER_MENTION% и %TIMEOUT%)
    │
    └─ kick_timeout > 0?
            ├─ да, timeout ≥ 10 → запланировать notify job   [notify_{chat_id}_{user_id}]
            └─ да             → запланировать kick job        [kick_{chat_id}_{user_id}]

Пользователь пишет #whois (≥20 символов)
    │
    ▼
on_hashtag_message()
    ├─ Записать в БД: User(chat_id, user_id, whois=текст)
    ├─ cancel_kick_jobs() → отменить таймеры и удалить приветственное сообщение
    └─ Если таймер был → ответить on_introduce_message

Таймер истёк (никто не представился)
    │
    ▼
on_kick_timeout()
    ├─ Удалить приветственное сообщение
    ├─ ban_chat_member(until_date=+60с)  ← временный кик
    └─ Отправить on_kick_message (если не "false" / "0")
```

## Настройка бота администратором

```
Администратор добавляет бота в чат
    │
    ▼
Администратор пишет в чат сообщение с #whois (≥20 символов)
→ Записывается в таблицу users
    │
    ▼
Администратор пишет боту в личку: /start
    │
    ▼
on_start_command()
    └─ Показывает список чатов, где пользователь есть в users И является admin/creator
    │
    ▼
Администратор выбирает чат → on_button_click(select_chat)
    └─ Показывает меню настроек (inline keyboard)
    │
    ▼
Администратор нажимает кнопку настройки → on_button_click(set_*)
    ├─ Проверка authorize_user (защита от подделки callback_data)
    ├─ context.user_data["action"] = Actions.set_*
    └─ "Отправьте новое значение"
    │
    ▼
Администратор отправляет текст → on_message() (DM ветка)
    ├─ Проверка authorize_user
    ├─ Обновить Chat в БД через sess.merge()
    └─ Показать кнопки "К настройке" / "К списку чатов"
```

## Ручные команды администратора

```
/skip (в ответ на сообщение)
    ├─ Проверить, что автор команды — admin
    └─ cancel_kick_jobs() для целевого пользователя
       └─ Пользователь НЕ записывается в БД → при следующем входе снова попросят представиться

/approve (в ответ на сообщение)
    ├─ Проверить, что автор команды — admin
    ├─ Записать User в БД (whois = "Одобрен администратором")
    └─ cancel_kick_jobs() для целевого пользователя
       └─ Пользователь записан → при следующем входе будет приветствоваться как знакомый
```

## Regex-фильтр

```
Любое сообщение в группе
    │
    ▼
on_message() или on_forward()
    ├─ Пользователь admin/creator? → пропустить
    ├─ chat.regex_filter is None?  → пропустить
    ├─ filter_only_new_users=True И пользователь уже в БД? → пропустить
    │
    └─ Совпадение с regex?
            ├─ Удалить сообщение
            ├─ cancel_kick_jobs()
            ├─ Отправить on_filtered_message
            └─ ban_chat_member(until_date=+60с)
```

## Жизненный цикл сообщений бота

Приветственное сообщение удаляется в трёх случаях:
1. Пользователь написал `#whois` — `cancel_kick_jobs()` удаляет его
2. Администратор выполнил `/skip` — то же
3. Таймер истёк — `on_kick_timeout()` удаляет перед киком

Сообщение-напоминание (`notify_message`) удаляется автоматически через `notify_delta` минут после отправки (джоб `delete_message`).
